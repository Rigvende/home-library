## 2. Major modules

### `src/lib/books.ts` — Domain layer

Pure functions and types; no React dependency.

| Export | Kind | Description |
|--------|------|-------------|
| `Book` | type | Full book record |
| `ReadingStatus` | type | `'unread' \| 'reading' \| 'read'` |
| `loadBooks()` | function | Reads + validates `localStorage` → `Book[]` |
| `saveBooks(books)` | function | Serialises `Book[]` → `localStorage` |
| `createBook(input)` | function | Stamps `id`, `addedAt`, `updatedAt` onto a new book |
| `updateBook(existing, patch)` | function | Merges patch, bumps `updatedAt` |

Internal helpers (not exported): `isBookLike` (type guard), `normalizeBook` (coerces partial data from storage).

Storage key: `homeLibrary.books.v1` in `localStorage`.

### `src/components/LibraryApp.tsx` — UI layer

Single `'use client'` component that owns all application state.

**Local types**

| Type | Description |
|------|-------------|
| `Draft` | Mutable form state (all fields as strings for controlled inputs) |

**State**

| Hook | Type | Purpose |
|------|------|---------|
| `useState<Book[]>` | `books` | Master book list, populated from `loadBooks()` after hydration |
| `useRef<boolean>` | `hydrated` | Guards the save effect from firing before the load effect |
| `useState<string>` | `query` | Live search string |
| `useState<Draft>` | `draft` | Current add/edit form values |
| `useState<string \| null>` | `editingId` | `null` = add mode; set = edit mode |

**Derived values (useMemo)**

| Value | Description |
|-------|-------------|
| `filtered` | Books sorted by `updatedAt` desc, filtered by `query` |
| `editingBook` | Resolved `Book` object for `editingId`, or `null` |

**Side effects**

| Hook | Trigger | Effect |
|------|---------|--------|
| `useEffect` (save) | `books` changes | Calls `saveBooks(books)` if `hydrated.current` is true |
| `useEffect` (load) | mount only | Calls `loadBooks()`, sets books, marks `hydrated.current = true` |

> The load effect runs after the save effect (definition order). This prevents the save effect from firing with an empty array before localStorage has been read.

**Event handlers**

| Handler | Description |
|---------|-------------|
| `resetForm()` | Clears draft, clears `editingId` |
| `startEdit(b)` | Populates draft from book, sets `editingId` |
| `remove(id)` | Filters book out of state; resets form if editing it |
| `toggleRead(id)` | Toggles status between `'read'` and `'unread'` |
| `onSubmit(e)` | Validates, creates or updates book, resets form |

**Sub-components**

| Component | Props | Description |
|-----------|-------|-------------|
| `Field` | `label`, `children` | Labelled form field wrapper |
| `StatusPill` | `status: ReadingStatus` | Coloured badge: emerald = read, amber = reading, zinc = unread |

## 4. Important relationships

### Import graph

```
src/app/page.tsx
  └── src/components/LibraryApp.tsx
        └── src/lib/books.ts

src/lib/__tests__/books.test.ts
  └── src/lib/books.ts

tests/e2e/library.spec.ts
  └── (browser: http://localhost:3000)
```

### Data flow

```
localStorage (homeLibrary.books.v1)
  │
  └── loadBooks()  ← useEffect on mount (after hydration)
        │
        ▼
  LibraryApp — useState<Book[]>(books)
        │
  user actions: add / edit / delete / toggleRead
        │
        └── saveBooks()  ← useEffect on [books] (guarded by hydrated ref)
              │
              ▼
        localStorage (homeLibrary.books.v1)
```

### State → UI mapping

```
books + query
  └── filtered (useMemo) → book list rows
editingId
  └── editingBook (useMemo) → form heading, Cancel button, submit label
draft
  └── controlled inputs in the add/edit form
```
